Design Evolution

From formative feedback

The Intersect-Assist alpha prototype is a revision of the application which improves upon some of the functionality and usability of the system. We reviewed the results and suggested changes from the Glass Hook team, following the feedback they provided after their user testing and heuristic evaluation. We have chosen, in the interest of time, to implement the most feasible suggestions for the alpha prototype; although other ones are relevant, they are not feasible for this project (this includes the development of an iOS application). Additionally, we decided on some design changes that we felt would meet our usability goals more effectively and improve the application’s usability score on Nielsen’s heuristics.

Compatibility with TalkBack

The Intersect assist team initially made an oversight in the computer prototype in not adhering to what the user population is accustomed to. It was assumed that the user’s would find the audio output of the system sufficient in completing the task. However, during the user testing performed by the Glass hook team, this posed an issue to the user and required them to disable the TalkBack option from settings. In response to this issue, we have made the new prototype compatible with TalkBack, an android-provided tool to facilitate navigation of the application for visually-impaired users. The addition of this feature is adheres to Nielsen’s Consistency and Standards heuristic.

Modification of Affordances and Addition of On-screen Information

Another addition to the design of the Intersect-Assist application is the display of affordances on the screen. This issue was reported by the Glass Hook team after the user complained that she wasn’t sure how to proceed from her current state. In that regard, we modified all the affordances of the system.

Specifically, we moved away from the use of gestures such as swipe up and swipe down, which relied on the user’s memory. Instead, we introduced buttons on screen which display the possible inputs the user can do. As an example, initially from the main menu, the user needed to swipe up to enter the mapping menu or swipe down to enter the veering and traffic light detection. We modified our system so that buttons are displayed on-screen with one button reading “Mapping Menu” and another reading “Light Detection and Veering”. With the use of TalkBack, we are confident that the addition of these buttons will provide the user with more control and project confidence in the user when performing a task.

Although this modification wasn’t explicitly pointed out by the Glass Hook team, we believe that the underlying cause of the problem the user encountered was the reliance on her memory for what each swiping direction accomplished. This modification adheres to Nielsen’s recognition rather than recall and user control and freedom heuristics.

Addition of Page Titles

Another modification to the system is the addition of titles to each page in the system. This is something that the Intersect-Assist team found was lacking from the application. Although this might seem trivial, the team was naive in thinking it wasn’t necessary for the specified user population. However, after careful review, it was established that since there are two submenus in the proposed system, providing a title to each page gives the user access to more information about the pages. And with the addition of TalkBack, the possibilities of user confusion will be greatly diminished. This modification adheres to Nielsen’s visibility of system status heuristic.

Providing the User with an End of Task prompt

One of the issues discovered during user testing with the Glass Hook team was the lack of a sense of “ending of a task” when the test subject was done crossing the street. However, since we don’t foresee our system having the capability of detecting when the user has crossed the intersection, we have implemented a button that the user presses when they have completed crossing the intersection. This button will read “I’m Done!” and will provide the user with a better sense of where they are in the application. This modification also adheres to Nielsen’s visibility of system status heuristic.

Refinement of Documentation

User Manual

The version 2 of our user manual contains many additions since our first user manual was greatly lacking. We added a table of content, installation guide and sections on navigating through the application. Since there was no comment made by the Glass Hook team regarding the minimalist user manual we provided for the computer prototype (Understandably so), the modifications we have done are solely on the team’s testing.

The first version of the user manual contained an introduction section and a brief tutorial on navigating through the application. However, there was a lack of task-based sections and there weren't any figures/images to help guide through the application. This new user manual includes figures which can help (but are not necessary) for understanding the flow of the application.
